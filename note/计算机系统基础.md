### 2.1.3 定点与浮点表示

#### 定点表示

定点表示法用来对定点小数和定点整数进行表示。对于定点小数，其小数点总是固定在数的左边，一般用来表示浮点数的尾数部分。对于定点整数，其小数点总是固定在数的最右边，因此可用“定点整数”来表示整数。

#### 浮点表示

对于任意一个实数$X$, 可以表示成如下形式:

$$
X = (-1)^s \times M \times R^E
$$

其中, $s$是符号位, $M$是尾数, $R$是基数, $E$是指数. 计算机系统中常用的浮点表示方法有IEEE 754标准, 该标准定义了单精度和双精度两种浮点数格式。

### 3.3.7 子程序

#### 1. 堆栈

(1) 相关概念

堆栈: 主存中的一片数据存贮区
压入: 将数据存入栈顶
弹出: 从栈顶取出数据
栈底: 栈中数据存储的起始位置 $EBP$
栈顶: 栈中数据存储的结束位置 $ESP$(栈顶指针)由高地址向低地址移动

栈底为大地址, 栈顶为小地址, 从小到大存储

![xt1](/media/xt1.png)

(2) 堆栈指令

a. 进栈指令

```assembly
PUSH OPS
```

OPS: 寄存器、段寄存器、存储器

b. 出栈指令

```assembly
POP OPD
```

OPD: 寄存器、段寄存器（除CS）、存储器

c. 8个寄存器进出栈指令

```assembly
PUSHA
```
将8个16位寄存器按AX、CX、DX、BX、SP、BP、SI、DI顺序入栈

d. 8个寄存器出栈指令

```assembly
POPA
```
将8个16位寄存器按DI、SI、BP、SP、BX、DX、CX、AX顺序出栈

#### 2. 子程序

(1) 相关概念

一个程序中常常有类似的程序段，这些程序段的功能和结构相同，只是某些变量的赋值不同。可以把这些程序段写成具有独立功能的模块的形式，以便需要时调用它。这些模块称为子程序。

(2) 子程序与主程序的关系——调用与返回

调用: 转移到子程序的第一行指令处，开始执行子程序的指令

返回: 转移到主程序中，调用指令的下一行指令处，开始执行主程序
的指令

实现方式: 借助堆栈，保存返回点的地址；利用出栈指令，实现子程序的返回。

(3) 子程序的定义

格式:

```assembly
.type func_name, @function
func_name:
# 子程序体
```

例：定义一个名为sort的子程序
```assembly
.type sort, @function
sort:
    ...
ret
```

(4) 子程序的调用和返回

a. 调用指令CALL

```assembly
CALL OPD
```
按照OPD的寻址方式,分为直接调用与间接调用

例:
```assembly
CALL sort
CALL 0x8048933
```

```assembly
MOVL $sort, var_1
CALL *var_1        *表示间接调用
```

```assembly
MOVL $sort, var_1
MOV $var_1, %eax
CALL *(%eax)         *表示间接调用,(%eax)的内容是sort的地址
```

b. 返回指令RET

```assembly
RET
```
RET指令的功能是从子程序返回到主程序中，转移到调用指令的下一行指令处，继续执行主程序的指令。

```assembly
RET n
```
出栈到EIP，并继续出栈n个字节。

注意: CALL和RET不影响标志位。

### 3.4.4 过程调用的机器级表示

#### 1. 过程调用的运行机理

##### (1). 主调函数 (Caller)
a. **压参**：将参数从右向左依次 `push` 入栈。
b. **调用**：执行 `call <func_addr>`，自动将返回地址 (EIP) 压栈，并跳转至被调函数。
c. **清理**：函数返回后，执行 `add $N, %esp` 清理栈上的参数（N = 参数总字节数）。
d. **取值**：从 `%eax` 寄存器中获取返回值。

##### (2). 被调函数 (Callee)
a. **保存现场**：
   - `push %ebp`：保存旧基址指针。
   - `mov %esp, %ebp`：建立新栈帧基址。
   - `sub $M, %esp`：为局部变量分配栈空间（M = 局部变量总大小）。
b. **执行逻辑**：通过 `0x8(%ebp)`、`0xc(%ebp)` 等偏移访问参数；通过 `-0x4(%ebp)` 等偏移访问局部变量。
c. **返回准备**：将返回值存入 `%eax`。
d. **恢复现场**：
   - `mov %ebp, %esp`：释放局部变量空间。
   - `pop %ebp`：恢复旧基址指针。
e. **返回**：执行 `ret`，弹出返回地址到 EIP，跳回主调函数。

![xt3](/media/xt3.png)

#### 2. 局部变量

- 子程序开始执行时，在堆栈中分配局部变量空间
- 局部变量在子程序的栈帧中
- 子程序退出前，会释放局部变量的空间
- 因此，C语言局部变量的作用域和生存空间，只在该子程序中

#### 3. 按值传递参数和按地址传递参数

##### (1) 形式参数和实际参数

- 形式参数: 函数定义, 只相当于一个占位符
- 实际参数: 函数调用, 提供真实数据

##### (2) 按值传递参数和按地址传递参数

- 按值传递参数: 主程序调用子程序时，传递的是一个变量的值。即将一个变量的值复制到了堆栈段中。子程序修改该参数时，只是修改了堆栈段中该参数的值，并没有修改主程序中原变量的值。

- 按地址传递参数：主程序调用子程序时，传递的是一个变量的值。即将一个变量的地址复制到了堆栈段中。子程序可以修改该地址寻址到的操作数的值，即可以修改主程序中原变量的值。
