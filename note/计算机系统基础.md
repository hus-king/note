## 2.1 数制与编码

在计算机内部, 数值数据的表示方法有两大类:第一种是直接用二进制表示; 另一种是采用二进制编码的十进制数(BCD码).

### 2.1.3 定点与浮点表示

#### 定点表示

定点表示法用来对定点小数和定点整数进行表示。对于定点小数，其小数点总是固定在数的左边，一般用来表示浮点数的尾数部分。对于定点整数，其小数点总是固定在数的最右边，因此可用“定点整数”来表示整数。

#### 浮点表示

对于任意一个实数$X$, 可以表示成如下形式:

$$
X = (-1)^s \times M \times R^E
$$

其中, $s$是符号位, $M$是尾数, $R$是基数, $E$是指数. 计算机系统中常用的浮点表示方法有IEEE 754标准, 该标准定义了单精度和双精度两种浮点数格式。

### 2.1.4 定点数的编码表示

定点数的编码表示方法有两种:原码和补码.

### 3.3.7 子程序

#### 1. 堆栈

(1) 相关概念

堆栈: 主存中的一片数据存贮区
压入: 将数据存入栈顶
弹出: 从栈顶取出数据
栈底: 栈中数据存储的起始位置 $EBP$
栈顶: 栈中数据存储的结束位置 $ESP$(栈顶指针)由高地址向低地址移动

栈底为大地址, 栈顶为小地址, 从小到大存储

![xt1](/media/xt1.png)

(2) 堆栈指令

a. 进栈指令

```assembly
PUSH OPS
```

OPS: 寄存器、段寄存器、存储器

b. 出栈指令

```assembly
POP OPD
```

OPD: 寄存器、段寄存器（除CS）、存储器

c. 8个寄存器进出栈指令

```assembly
PUSHA
```
将8个16位寄存器按AX、CX、DX、BX、SP、BP、SI、DI顺序入栈

d. 8个寄存器出栈指令

```assembly
POPA
```
将8个16位寄存器按DI、SI、BP、SP、BX、DX、CX、AX顺序出栈

#### 2. 子程序

(1) 相关概念

一个程序中常常有类似的程序段，这些程序段的功能和结构相同，只是某些变量的赋值不同。可以把这些程序段写成具有独立功能的模块的形式，以便需要时调用它。这些模块称为子程序。

(2) 子程序与主程序的关系——调用与返回

调用: 转移到子程序的第一行指令处，开始执行子程序的指令

返回: 转移到主程序中，调用指令的下一行指令处，开始执行主程序
的指令

实现方式: 借助堆栈，保存返回点的地址；利用出栈指令，实现子程序的返回。

(3) 子程序的定义

格式:

```assembly
.type func_name, @function
func_name:
# 子程序体
```

例：定义一个名为sort的子程序
```assembly
.type sort, @function
sort:
    ...
ret
```

(4) 子程序的调用和返回

a. 调用指令CALL

```assembly
CALL OPD
```
按照OPD的寻址方式,分为直接调用与间接调用

例:
```assembly
CALL sort
CALL 0x8048933
```

```assembly
MOVL $sort, var_1
CALL *var_1        *表示间接调用
```

```assembly
MOVL $sort, var_1
MOV $var_1, %eax
CALL *(%eax)         *表示间接调用,(%eax)的内容是sort的地址
```

b. 返回指令RET

```assembly
RET
```
RET指令的功能是从子程序返回到主程序中，转移到调用指令的下一行指令处，继续执行主程序的指令。

```assembly
RET n
```
出栈到EIP，并继续出栈n个字节。

注意: CALL和RET不影响标志位。

(5) 现场保护和现场恢复

```assembly
# protect.s
f_1:
push %eax
push %ebx
… ; 子程序体
pop %ebx
pop %eax
ret
```

注意: 若使用寄存器EAX存放子程序的返回值，则不能对其进行保护和恢复。
