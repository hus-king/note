## 3.1 SQL概述

- 综合统一:集数据定义语言（DDL），数据操纵语言（DML），数据控制语言（DCL）功能于一体。
- 高度非过程化
- 面向集合的操作方式
- 以同一种语法结构提供多种使用方式
- 语言简洁，易学易用————9个动词:(查询)select,(定义)create,drop,alter,(操纵)delete,update,insert,(控制)grant,revoke

## 3.2 数据定义

建立数据库时首先要对数据库中数据的结构及特征进行描述，该过程称为数据定义.

在SQL语言中，关于数据的描述信息以SQL模式（Schema）的形式组织。每个模式包含0到多个对象，这些对象被称为模式对象，如基表、视图、索引等。

SQL模式语句可分为三类：CREATE语句、DROP语句和ALTER语句。

### 表：SQL 的数据定义语句

| 操作对象 | 创建 (CREATE) | 删除 (DROP) | 修改 (ALTER) |
| :--- | :--- | :--- | :--- |
| **模式 (SCHEMA)** | `CREATE SCHEMA` | `DROP SCHEMA` | |
| **表 (TABLE)** | `CREATE TABLE` | `DROP TABLE` | `ALTER TABLE` |
| **视图 (VIEW)** | `CREATE VIEW` | `DROP VIEW` | |
| **索引 (INDEX)** | `CREATE INDEX` | `DROP INDEX` | `ALTER INDEX` |

### 3.2.1 模式的定义与删除

#### 1. 模式的定义

模式定义语句如下:

```sql
CREATE SCHEMA <模式名> AUTHORIZATION <用户名>
    [<表定义子句>|<视图定义子句>|<授权定义子句>];
```

语义: 为某用户创建一个模式。定义模式实际上定义了一个命名空间，其中可以定义该模式包含的数据库对象，例如基本表、视图、索引等。

#### 2. 模式的删除

模式删除语句如下:

```sql
DROP SCHEMA <模式名> <RESTRICT|CASCADE>;
```

语义: 删除一个模式。删除模式时，如果该模式中存在对象，则必须指定删除方式：<RESTRICT> 或 <CASCADE>
- <RESTRICT>(限制): 如果模式中存在对象，则拒绝删除模式。
- <CASCADE>(级联): 删除模式及其包含的所有对象。

### 3.2.2 基本表的定义、删除与修改

#### 1. 基本表的定义

基本表定义语句如下:

```sql
CREATE TABLE <表名> (
    <列名><数据类型>[列级完整性约束条件]
    [,<列名><数据类型>[列级完整性约束条件]]
    [,PRIMARY KEY(列名[,列名]...)]
    [,FOREIGN KEY(列名[,列名]...) REFERENCES <表名>(列名[,列名]...)]
    [,UNIQUE(列名[,列名]...)]
    [,CHECK(<条件>)]
    ...
    [,<表级完整性约束条件>]
);
```

语义: 定义一个基本表。定义基本表时，必须指定表名和至少一个列定义。每个列定义包括列名、数据类型和可选的列级完整性约束条件。还可以指定表级完整性约束条件。

- <表名>: 所要定义的基本表的名字
- <列名>: 组成该表的各个属性（列）
- <列级完整性约束条件>：涉及相应属性列的完整性约束条件
- <表级完整性约束条件>：涉及一个或多个属性列的完整性约束条件
- PRIMARY KEY: 主键约束，指定一个或多个列作为主键
- FOREIGN KEY: 外键约束，指定一个或多个列作为外键，并引用另一个表的主键
- UNIQUE: 唯一约束，指定一个或多个列的值必须唯一
- CHECK: 检查约束，指定一个条件，要求列的值必须满足该条件

例子:
```sql
CREATE TABLE Course (
    Cno CHAR(4) PRIMARY KEY,          -- 课程号：主码，固定长度4字符
    Cname CHAR(40) NOT NULL,          -- 课程名：非空，固定长度40字符
    Cpno CHAR(4),                     -- 先修课号：允许为空（即没有先修课）
    Ccredit SMALLINT,                 -- 学分：使用小整数类型
    
    /* 表级完整性约束：
       定义外码 Cpno 参照本表 Course 的主码 Cno。
       这是一种“自引用”结构，表示先修课必须是课程表中已有的课程。
    */
    FOREIGN KEY (Cpno) REFERENCES Course(Cno)
);
```
#### 2. 数据类型

##### SQL92 数据类型及其含义

| 数据类型 | SQL92 数据类型 - 含义 |
| :--- | :--- |
| **CHAR(n)** | 长度为 $n$ 的定长字符串 |
| **VARCHAR(n)** | 最大长度为 $n$ 的变长字符串 |
| **INT** | 长整数（也可以写作 INTEGER），范围：$-2^{31} \sim 2^{31} - 1$ |
| **SMALLINT** | 短整数，范围：$-2^{15} \sim 2^{15} - 1$，占用 2 字节 |
| **NUMERIC(p,d)** | 定点数，由 $p$ 位数字（不包括符号、小数点）组成，小数点后面有 $d$ 位数字 |
| **REAL** | 取决于机器精度的单精度浮点数 |
| **Double Precision** | 取决于机器精度的双精度浮点数，占用 8 字节 |
| **FLOAT(n)** | 可选精度的浮点数，精度至少为 $n$ 位数字，占用 4 字节 |
| **DATE** | 日期，包含年、月、日，格式为 `YYYY-MM-DD` |
| **TIME** | 时间，包含一日的时、分、秒，格式为 `HH:MM:SS` |
| **INTERVAL** | 两个 date 或 time 类型数据之间的差 |

#### 3. 模式与表

每一个基本表都属于某一个模式
一个模式包含多个基本表
定义基本表所属模式（常用方法2种）：
- 在表名中明显地给出模式名，如`Create table “S-T”
.Student（......）;`
- 在创建模式语句中同时创建表，如
```sql
CREATE database School;
use School;
CREATE TABLE Student (...);
```

#### 4. 修改数据表

一般格式：
```sql
ALTER TABLE <表名>
    [ ADD <新列名> <数据类型> [ 完整性约束 ] ]
    [ ADD <表级完整性约束> ]
    [ DROP [COLUMN] <列名> [CASCADE｜RESTRICT]]
    [ DROP CONSTRAINT<完整性约束名>[CASCADE｜RESTRICT] ]
    [ ALTER COLUMN<列名> TYPE <数据类型> ]
    [ RENAME COLUMN <列名> TO <新列名>]；
```
语义: 修改一个基本表的结构。可以添加或删除列，添加或删除完整性约束，修改列的数据类型，重命名列等。
- ADD: 添加新列或表级完整性约束
- DROP: 删除列或完整性约束
- ALTER COLUMN: 修改列的数据类型
- RENAME COLUMN: 重命名列

#### 5. 删除数据表

```sql
DROP TABLE <表名> [RESTRICT|CASCADE];
```

### 3.2.3 索引的建立与删除

#### 1. 索引的建立

```sql
CREATE [UNIQUE] [CLUSTER] INDEX <索引名>
    ON <表名>(<列名>[<次序>][,<列名>[<次序>] ]…)；
```

其中，<次序> 可以是 ASC（升序）或 DESC（降序），默认是 ASC。

功能: 在<表名>的表上，对其中的指定列，建立一个名为<索引名>的索引文件。

作用: 提高查询速度

说明:
- UNIQUE(单一索引): 指定索引列的值必须唯一
- CLUSTER(聚集索引): 索引项的顺序与表中记录的物理顺序一致。表中如果有多个记录在索引字段上相同，这些记录构成一簇，只有一个索引值。

#### 2. 索引的修改

```sql
ALTER INDEX <索引名> RENAME TO <新索引名>;
```

#### 3. 索引的删除

```sql
DROP INDEX <索引名>；
或
DROP INDEX <索引名> ON <表名>；
```
