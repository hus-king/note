## 2.4 数据信息的校验

### 2.4.1 码距与校验

#### 定义

码距: 任意两个合法编码间不同的二进制位数

码距越大, 抗干扰能力越强, 纠错能力越强, 数据冗余越大, 编码效率越低

奇偶校验码最小码距为2, 海明码最小码距为3

#### 码距与纠错性能

- **`t` 是纠错的半径**，保证了错误码字不会和其他码字混淆。
- **`e` 是额外检测的宽度**，保证了错误码字在超出纠错范围后，依然能被识别为“非法”。
- **总检测能力 `t+e` 是两者之和**，超过这个范围，错误码字就可能和其他合法码字混淆，无法可靠处理。

当编码仅检错时, 码距 $d_{min} = e + 1$

当编码仅纠错时, 码距 $d_{min} = 2t + 1$

当编码既纠错又检错时, 码距 $d_{min} = 2t + e + 1$

### 2.4.2 奇偶校验

- **奇校验**：使得整个码字中1的个数为奇数
- **偶校验**：使得整个码字中1的个数为偶数

冗余位: 1位 校验位P
最小码距: 2

#### 二维奇偶校验

- 若干数据一个校验位

- 整个数据包增加一个校验字节

- ALL 3-bit errors

- Most 4-bit errors

### 2.4.3 海明校验

#### 编码规则

- $(n,k)$ 海明码共 $n$ 位, 其中 $k$ 位为数据位,  $r$ 位为校验位, 满足 $ 2^r - 1\geq k + r = n $

数据位: $ D_4 D_3 D_2 D_1 $

校验位: $ P_1 P_2 P_3 $

映射:
| 位置 | $H_1$ | $H_2$ | $H_3$ | $H_4$ | $H_5$ | $H_6$ | $H_7$ |
|--------|-----|-----|-----|-----|-----|-----|-----|
| 类型 | $P_1$ | $P_2$ | $D_1$ | $P_3$ | $D_2$ | $D_3$ | $D_4$ |

检错码:

$ G_1 = H_1 \oplus H_3 \oplus H_5 \oplus H_7 = P_1 \oplus D_1 \oplus D_2 \oplus D_4 $

$ G_2 = H_2 \oplus H_3 \oplus H_6 \oplus H_7 = P_2 \oplus D_1 \oplus D_3 \oplus D_4 $

$ G_3 = H_4 \oplus H_5 \oplus H_6 \oplus H_7 = P_3 \oplus D_2 \oplus D_3 \oplus D_4 $

### 2.4.4 CRC循环冗余校验

#### 编码规则

- 编码可被生成多项式整除

- 设CRC码 $n$ 位, 其中数据位 $k$ 位, 校验位 $r$ 位, 满足 $ n = k + r \leq 2^r - 1 $

原数据表示为 $M(x)$, 生成多项式为 $G(x)$, 商为 $Q(x)$ , 余数为 $R(x)$, 则有
$$
M(x) \cdot x^r + R(x) = Q(x) \cdot G(x) + R(x) + R(x) = Q(x) \cdot G(x)
$$

#### 生成多项式

- $G(x) = x + 1 = 11$
    - (7,6)码, 判一位错
- $G(x) = x^3 + x + 1 = 1011$
    - (7,4)码, 判两位错 或者 纠一位错
    - 两位错,一位错余数均不为零,但余数有重叠
- $G(x) = (x+1)(x^3+x+1) = 11101$
    - (7,3)码, 判两位错 并且 纠一位错
    - 两位错,一位错余数均不为零,且余数无重叠

### 2.4.5 三种校验码的比较

|              | 偶校验                     | 可检一位错海明码                | CRC循环冗余校验码                  |
| ------------ | -------------------------- | ------------------------------- | --------------------------------- |
| 冗余信息     | $r=1$                      | $r>1$（交叉奇偶校验）           | $r>1$（交叉奇偶校验）             |

| 校验码公式   | $\boldsymbol{P = D_1\oplus D_2 \dots \oplus D_k}$ | $\boldsymbol{P_i = H_?\oplus H_? \oplus\dots H_?}$ | $\boldsymbol{P = (M(x)\cdot 2^r)\% G(x)}$ |

| 检错依据     | $\boldsymbol{G=p\oplus d_1\oplus d_2 \dots \oplus d_k=0}$ | $\boldsymbol{G_r\dots G_2G_1 = 0}$ | $\boldsymbol{R(x) = 0}$            |

### 3.2.2 溢出及检测

#### 1. 单符号

法一:设两数符号位为$f_0$和$f_1$,和符号位$f_s$,则

$$
OF=\overline{f_0}\cdot \overline{f_1} \cdot f_s + f_0 \cdot f_1 \cdot \overline{f_s}
$$

法二:符号位进位为$C_f$, 最高位进位为$C_n$, 则

$$
OF = C_f \oplus C_n
$$

#### 2. 双符号

双符号位最高位永远是正确符号位

01为正溢出,10为负溢出

$$
OF = f_1 \oplus f_2
$$

## 4.5 高速缓冲存储器——Cache

### 4.5.1 cache工作原理

早期U盘写速度慢，部分内存空间为U盘作写缓冲, 数据写入内存即报告完成

- 可有效提升写性能，改善用户等待体验
- 导致数据不一致性，产生脏数据---还未写入

快存为慢存作写缓冲优化写性能

### 4.5.2 程序局部性

![zy24](./media/zy24.png)

程序局部性————程序仅需访问内存很小一部分空间
- 时间局部性: 程序倾向于访问最近访问过的内存位置
- 空间局部性: 程序倾向于访问与最近访问过的内存位置相邻的内存位置

优化手段
- 预读优化: 空间局部性
- 调度算法: 时间局部性

cache基本思想: 在处理器附近增加一个隐藏的小容量快速存储器
- 对程序员透明(可感知, 无法操控)
- 将经常访问的热数据、即将访问的数据放入cache中, 提高访问速度
- 提高命中率优化读性能


### 4.5.3 cache的基本概念

在处理器附近增加一个隐藏的小容量快速存储器
- 对程序员透明(可感知, 无法操控)
- 将经常访问的热数据、即将访问的数据放入cache中, 提高访问速度
- 提高命中率优化读性能

cache术语

- 命中: CPU访问数据在cache中
- 缺失: CPU访问数据不在cache中, 需要从主存调入cache
- 块: cache与主存交换最小单位
- 行/槽: 标记、标志位、数据块容器
- 命中率: 主存访问中cache命中的比例
- 缺失率: 1 - 命中率
- 命中访问时间: 数据查找时间、cache访问时间、总线传输时间
- 缺失损失: 主存块调用cache, 数据传输到处理器的时间

#### 3. cache关键技术

- 数据查找

    - 相联存储器
        - 按内容进行访问(key, value)
        - 以关键字作全局并发比较, 硬件成本高
        - 片外cache用于存放查找表
        - 片内cache存放查找表+数据副本
        - 虚拟存储器中存放段表、页表

    - CPU cache 基本组织方式
        - CPU cache由SRAM构成
        - cache与主存均分为固定大小的数据块, 以块为单位交换数据
        - 相联存储器存放查找表/cache
        - CPU给出的块地址与查找表中某单元相同且有效位为1时表示命中


- 地址映射

    - 全相联
        ![zy33](./media/zy33.png)
        - 块映射灵活, 一对多映射
        - cache满后才会出现块冲突
        - 块冲突概率低, cache利用率高
        - 淘汰算法复杂
        - 命中率高

    - 直接相联

        ![zy31](./media/zy31.png)
        - 映射速度快, 一对一映射, 无须查表
            - 利用索引字段直接对比相应标记位即可
            - 查找表可以和副本一起存放，无需相联存储器
            - 查找时间和全相联一样（一级比较器延迟）
        - 块冲突概率高, cache利用率低
        - 淘汰算法简单, 直接淘汰
        - 命中率低
    - 组相联
        ![zy32](./media/zy32.png)
        - 多个相联存储器共享一个多路比较器
        - 相对于全相联 多路比较器复杂度低

### 4.6.1 虛拟存储器工作原理

页式虚拟地址空间: 虚拟地址空间被划分为固定大小的页, 主存被划分为同样大小的页框, 虚拟页与物理页框之间通过页表进行映射

- 程序访问虚拟地址时, 通过页表将虚拟地址转换为物理地址, 实现地址映射

- 操作系统负责将数据从磁盘迁移到内存
- 当前进程挂起
- 操作系统负责所有的替换策略
- 唤醒挂起进程

#### 虚拟存储器本质

- 程序执行时，按需载入代码和数据（节省空间）

#### 与CPU cache的相似之处

- 将程序中常用的部分驻留在高速存储器

    - 高速存储器分块或者分页（粒度问题）

    - 主存空间满，将不常用程序或数据淘汰或交换到辅存中

- 数据的换入换出由硬件或操作系统完成，对用户透明

- 充分利用程序局部性

    - 使存储系统的性能接近于高速存储器

    - 价格接近于低速存储器，并扩充主存容量

#### 与CPU cache的区别

- 虚存用于扩大主存容量，cache用于加速主存性能

- 虚存中未命中性能损失远大于cache系统

    - 全相联提升命中率

    - 更大的交换单位页

    - 近似LRU算法（CLOCK算法）

- 虚存由硬件和OS联合管埋，CACHE由硬件管理

### 4.6.3 页式虚拟存储器

#### 名词解释

- VA: Virtual Address 虚拟地址
    程序编译、指令寻址用的逻辑地址，CPU 访存首先给出 VA，不能直接去内存寻址，必须经 MMU 分页转换为物理地址 PA。

- VPN: Virtual Page Number 虚拟页号
    VA 的高位部分，用来索引页表、找 PTE；低位是页内偏移（页内地址不变，虚实偏移相同）。

- PTE: Page Table Entry 页表项
    存放在主存页表里的表项，是 VPN→PPN 的映射条目
    - 内容：PPN（物理页号）+ 标志位（有效位、访问权限位、脏位等）
    - 作用：指示虚拟页是否在内存中，若在内存中，提供物理页号；若不在内存中，触发缺页异常

- PTEA: Page Table Entry Address 页表项地址
    PTE 在内存中的地址，页表基址 + VPN * PTE 大小

- PPN: Physical Page Number 物理页框号
    物理地址的高位部分，页内偏移不变；PA = PPN * 页大小 + 页内偏移

- PPO: Page Offset 页内偏移
    VA 和 PA 的低位部分，页内地址不变

- PA: Physical Address 物理地址
    MMU 将 VA 转换为 PA 后，CPU 用 PA 访问内存

## 5.0 MIPS指令

### 5.0.1 MIPS概述

MIPS(Microprocessor without Interlocked Pipeline Stages)是一种典型的**RISC(精简指令集计算机)** 架构，具有以下特点：

- **指令长度固定**：所有指令均为32位，简化了指令译码
- **寻址方式简单**：只有有限的几种寻址方式
- **寄存器丰富**：32个通用寄存器(`$0`~`$31`)
- **Load-Store架构**：只有load/store指令能访问内存，运算指令只能操作寄存器
- **流水线友好**：指令格式规整，便于流水线实现

### 5.0.2 MIPS寄存器约定

| 寄存器 | 名称 | 用途 |
|--------|------|------|
| `$0` | `$zero` | 恒为0 |
| `$1` | `$at` | 汇编器保留 |
| `$2`~`$3` | `$v0`~`$v1` | 函数返回值 |
| `$4`~`$7` | `$a0`~`$a3` | 函数参数 |
| `$8`~`$15` | `$t0`~`$t7` | 临时寄存器 |
| `$16`~`$23` | `$s0`~`$s7` | 保存寄存器 |
| `$24`~`$25` | `$t8`~`$t9` | 临时寄存器 |
| `$28` | `$gp` | 全局指针 |
| `$29` | `$sp` | 栈指针 |
| `$30` | `$fp` | 帧指针 |
| `$31` | `$ra` | 返回地址 |

### 5.0.3 关键MIPS汇编指令

#### 一、算术运算指令（R型）

| 指令 | 示例 | 说明 |
|------|------|------|
| `add` | `add $rd, $rs, $rt` | `$rd = $rs + $rt`，溢出触发异常 |
| `addu` | `addu $rd, $rs, $rt` | 无符号加，不检测溢出 |
| `sub` | `sub $rd, $rs, $rt` | `$rd = $rs - $rt` |
| `subu` | `subu $rd, $rs, $rt` | 无符号减 |
| `addi` | `addi $rt, $rs, imm` | **I型**，`$rt = $rs + imm(符号扩展)` |
| `addiu` | `addiu $rt, $rs, imm` | 无符号立即数加 |
| `mul` | `mul $rd, $rs, $rt` | `$rd = $rs * $rt`（取低32位） |
| `div` | `div $rs, $rt` | 有符号除法，商→`$lo`，余→`$hi` |
| `divu` | `divu $rs, $rt` | 无符号除法 |
| `mfhi` | `mfhi $rd` | 将`$hi`复制到`$rd` |
| `mflo` | `mflo $rd` | 将`$lo`复制到`$rd` |

#### 二、逻辑运算指令（R型）

| 指令 | 示例 | 说明 |
|------|------|------|
| `and` | `and $rd, $rs, $rt` | `$rd = $rs & $rt`（按位与） |

| `nor` | `nor $rd, $rs, $rt` | `$rd = ~($rs \| $rt)`（按位或非） |
| `andi` | `andi $rt, $rs, imm` | **I型**，立即数与（零扩展） |
| `ori` | `ori $rt, $rs, imm` | 立即数或（零扩展） |
| `xori` | `xori $rt, $rs, imm` | 立即数异或（零扩展） |

#### 三、移位指令（R型）

| 指令 | 示例 | 说明 |
|------|------|------|
| `sll` | `sll $rd, $rt, shamt` | 左移，空位补0 |
| `srl` | `srl $rd, $rt, shamt` | 逻辑右移，空位补0 |
| `sra` | `sra $rd, $rt, shamt` | 算术右移，空位补符号位 |
| `sllv` | `sllv $rd, $rt, $rs` | 变量左移（移位量在`$rs`中） |

#### 四、数据传输指令（I型）

| 指令 | 示例 | 说明 |
|------|------|------|
| `lw` | `lw $rt, imm($rs)` | Load Word，从内存加载32位字到寄存器 |
| `sw` | `sw $rt, imm($rs)` | Store Word，将寄存器数据存入内存 |

## 5.3 寻址方式

### 5.3.1 指令寻址

#### 顺序寻址

- 程序对应的机器指令序列在主存顺序存放

- 执行时从第一条指令开始，依次执行后续指令

- 程序计数器(PC)对指令序号进行计数

- 执行一条指令, $PC = PC + $ 当前指令长度

#### 跳跃寻址

- 跳跃寻址方式: 当指令中出现分支或者循环时, 就会改变程序的执行顺序

- 跳跃的处理方式是重新修改程序计数器(PC)的值, 使其指向新的指令地址

### 5.3.2 操作数寻址

#### 立即寻址

- 地址码字段是操作数本身

#### 寄存器寻址

- 操作数在CPU的内部寄存器中

#### 直接寻址

- 地址码字段直接给出操作数在内存的地址

#### 间接寻址

- 地址码字段是操作数地址的地址

#### 寄存器间接寻址

- 地址码字段是寄存器编号, 寄存器中存放操作数地址

#### 相对寻址

- 地址码字段是一个偏移量, 以当前指令地址为基准进行寻址

#### 基址/变址寻址

操作数地址为 基址寄存器 + 变址寄存器 + 偏移量

#### 堆栈寻址方式

- 操作数在堆栈中, 通过栈指针寄存器寻址

# 第九章 输入输出原理

## 9.1 输入输出设备与特性

- 输入输出设备: 计算机与人或者机器系统进行数据交互的装置，用于实现计算机内部二进制信息与外部不同形式信息的转换，简称外部设备或外设

- 外部设备、接口部件、总线以及相应的管理软件统称I/O系统

## 9.2 I/0接口

- I/O接口: 连接总线与I/O设备的物理和逻辑界面

- I/O接口的功能:

    - 设备寻址、数据交互、设备控制
    - 状态检测、数据缓冲、格式转换

#### I/O接口结构

![zy131](./media/zy131.png)

#### I/O接口编址

- 统一编址

    - 内存映射编址

    - 外设地址与内存地址统一编址, 同一个地址空间

    - 不需要设置专用的I/O指令

    - 访存指令访问外设，访问什么设备取决于地址

- 独立编址

    - 端口映射编址

    - I/O地址空间与主存地址空间相互独立

    - I/O地址又称为I/O端口

    - 不同设备中的不同寄存器和存储器都有唯一的端口地址

    - 使用I/O指令访问外设
